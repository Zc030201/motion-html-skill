## 最终页面判定 v1

### 01 封面

- 核心判断：这不是 AI 技术细节课，而是一堂部门共同认知课。
- 保留内容：标题、适合不了解 AI 的部门学习、基于刘嘉《通用人工智能》、看懂/会用/会判断。
- 视觉判定：封面必须有最高级感，但不要复杂。左侧或中央大标题，右侧可用抽象 AI 光网/数据轨道。
- 动效判定：背景粒子慢流动即可，关键词按 `看懂 -> 会用 -> 会判断` 依次出现。
- 不要做：不要加卡通机器人，不要做复杂 3D，不要让封面变成炫技页。
- HTML 状态：保持现有方向，后续只做精修。

### 02 学习目标

- 核心判断：先告诉听众“学完要回答哪四个问题”。
- 保留内容：AI/ChatGPT/AGI 是什么；为什么 ChatGPT 不是普通软件；AI 能做什么；使用 AI 注意什么。
- 视觉判定：四张问题卡是正确结构，每张卡要有清晰问题，不要过度解释。
- 动效判定：卡片依次淡入，hover 时轻微抬升即可。
- 不要做：不要展开理论，不要加入 Agent 细节；Agent 在第 3/10 页讲。
- HTML 状态：内容已基本对齐，可优化卡片图标。

### 03 身边的 AI

- 核心判断：AI 不是新东西，变化在于从“专用系统”走向“大模型/Agent”。
- 保留内容：人脸识别、推荐算法、语音助手、地图导航、智能客服；ChatGPT、Claude、DeepSeek、豆包/文心、Agent。
- 视觉判定：采用 `过去的 AI -> 能力迁移 -> 现在的大模型` 三段式，而不是普通列表。
- 动效判定：中央能力迁移圆环慢速旋转，右侧 Agent 行用轻微紫色强调“执行”。
- 不要做：不要讲技术原理；本页只降低门槛、建立熟悉感。
- HTML 状态：已按此方向重做。

### 04 传统 AI vs 大模型

- 核心判断：差异不是“新旧软件”，而是能否跨任务、理解上下文、适应开放环境。
- 保留内容：特定任务/多任务、固定规则/上下文调整、封闭场景/开放场景、工具/助手。
- 视觉判定：能力矩阵优于左右普通表格，维度包括 `TASK / CONTEXT / ENVIRONMENT / ROLE`。
- 动效判定：矩阵行逐行出现，hover 高亮当前维度。
- 不要做：不要把 AGI 讲得过满；只说“大模型更接近 AGI 方向”。
- HTML 状态：已按矩阵重做。

### 05 大语言模型

- 核心判断：大模型不是搜索引擎，而是语言生成与理解系统。
- 保留内容：学习大量文本/代码/知识材料；根据上下文预测和生成；能写改总结翻译分析；不等于正确。
- 视觉判定：横向流程是对的，但节点要足够大，中央 `LLM` 核心表达“生成系统”。
- 动效判定：输入 -> 上下文 -> 生成 -> 判断按顺序点亮，中央 LLM 呼吸。
- 不要做：不要加入数据库/搜索引擎复杂架构图，避免越讲越技术。
- HTML 状态：现有大节点方案保留。

### 06 语言为什么关键

- 核心判断：语言是人类知识的压缩包，也是通向 AGI 的重要路径。
- 保留内容：知识、思想、协作、文明、AI。
- 视觉判定：中心 `语言` + 五个概念围绕，适合交互拖拽。
- 动效判定：五个概念慢速小范围漂移，可拖动，线条实时连接中心。
- 不要做：不要把每个概念解释太长；本页靠关系图表达。

### 07 为什么 AI 突然变强

- 核心判断：大模型的“大”重要，是因为数据、参数、算力、训练方法共同推动涌现。
- 保留内容：更多数据、更多参数、更强算力、更好训练方法、涌现；字典 -> 图书馆 -> 智库。
- 视觉判定：能力跃迁曲线比 3D 模型更合适。左侧三条驱动因素，右侧曲线穿过“涌现阈值”。
- 动效判定：曲线绘制、阈值发光、三个阶段标签逐步出现。
- 不要做：不要放复杂 3D 曲面；它会削弱“涌现”这个讲法。
- HTML 状态：已改为曲线方案，后续可精修阶段标签。

### 08 训练和教育

- 核心判断：从基础模型到可用助手，靠的是训练、提示、人类反馈和对齐。
- 保留内容：预训练、提示词、人类反馈、对齐、可用助手。

- 动效判定：阶梯底线逐级填充，五个阶段依次进入。

### 10 正确使用 AI / Agent 工作流

- 核心判断：AI 使用不是问一句，而是人类目标 + Agent 执行 + 人工检查的闭环。
- 保留内容：明确目标、提供背景、生成初稿、人工检查、继续修改、最终版本。
- 现代补充：Agent 可以规划步骤、调用工具、形成草稿，但不接管判断。
- 视觉判定：采用双层工作流：外层 `HUMAN GUARDRAILS`，内层 `AGENT EXECUTION LAYER`。
- 动效判定：环形路径绘制，Agent 核心呼吸，节点依次出现。
- 不要做：不要把 Agent 讲成全自动替代人；重点是“执行层”和“把关层”分开。
- HTML 状态：已按此方向重做，是后续主工作流标准。

## 全局设计判定

- 主题：浅色科技、部门培训、咨询公司/发布会混合风。
- 字体：中文优先 `Microsoft YaHei` / `Source Han Sans SC`，标题极粗，正文克制。
- 色彩：浅蓝灰背景、深蓝文字、电光蓝强调、少量青色和紫色用于 Agent/涌现。
- 卡片：统一玻璃质感，圆角不超过 28px，阴影轻，边框细。
- 动效节奏：每页 600-900ms 入场，复杂页面最多一个持续运动核心。
- 交互使用边界：只有 06、10、12 必须交互；07 是动效图表；其他页以稳定讲解为主。
- 导航：继续保持左右半屏点击翻页、键盘翻页、右下角进度条。
- 打包策略：最终确认前不打包；确认后再做单 HTML/资源包两种交付选项。
