Language Model

不是搜索引擎，而是语言生成与理解系统

它学习大量文本、代码和知识材料；根据上下文预测并生成回答。重点是“生成”，不是从数据库复制标准答案。

LANGUAGE ENGINE不是搜索引擎，而是语言生成与理解系统。
输入问题明确目标、背景、限制条件和输出要求。
理解上下文识别任务意图、语境关系和隐含约束。
生成回答写作、改写、总结、翻译、分析都属于生成能力。
人工判断它不等于一定正确，关键事实仍需人工核验；也可以让 AI 之间互相检查，再由人最终确认。

11 / 24
为什么可以更换 API

换 API，本质是换背后的模型服务

如果工具支持相近的接口格式，就可能把同一个工具连接到不同模型服务。界面可能不变，但能力、速度、价格、稳定性都会变。

工具是外壳你看到的是 Codex、Claude Code、OpenClaw 这类工作台；它们承载文件、权限和操作流程。
模型是大脑背后可以接不同模型服务，换模型后推理、速度、价格和稳定性都会变化。
API 是通道API Key 像身份卡，决定工具能连接谁、用多少、是否有权限调用。

Agentic AI

Agent：从“回答问题”到“推进任务”

Agent 不是一个新按钮，而是一种工作方式：模型会围绕目标做计划、调用工具、记住上下文，并把多步骤任务推进到可检查的结果。

有目标不是只接一句话，而是围绕一个任务持续推进。
会用工具可以检索资料、读文件、运行代码、调用系统或业务接口。
有反馈回路生成、检查、修正、再执行，直到得到更可用的版本。

Agent 的关键，是把任务一步步推进到结果

03
调用工具检索、读文件、写代码、操作网页或连接内部系统。
04
检查与修正对照目标修正输出，把过程留下可追踪记录。

AI 的风险

AI 不是权威答案，而是高效助手

幻觉像答得很自信，但可能把不存在的内容说成事实。

SAFE AI USE
22 / 24
安全不是不用文件，而是分级使用

安全用 AI：分级、脱敏、授权、留痕

不能简单理解成“所有文件都不能上传”。真正可落地的做法，是按数据级别、工具环境和任务风险来决定能给什么、怎么给、谁确认。

分级区分公开资料、内部普通资料、敏感资料和高风险资料。
脱敏去掉姓名、手机号、合同编号、金额明细等不必要敏感字段。
授权只给当前任务需要的文件和权限，高风险动作必须人工确认。
留痕记录输入材料、工具调用、输出版本和最终验收人。
可以用公开材料、脱敏样例、非敏感草稿、内部允许使用的知识库。
谨慎用内部文档、财务数据、客户材料，需要环境、权限和审批。
不能直接用密码、API Key、个人隐私、合同原文、高风险审批动作。

能用

USE AS ASSISTANT
